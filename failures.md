# Appendix for "Detecting Hidden Failures of DBMS: A Comprehensive Metamorphic Relation Output Patterns Approach" Published in IEEE COMPSAC 2023 Proceedings

This appendix details a list of observed logic failures in the OcaseBase v3.1.0 Community Edition. These failures serve as practical examples of the concepts discussed in the main paper. Each failure is categorized, described, and accompanied with a test case code snippet for reproduction. The failures are identified using the Metamorphic Relationship Output Pattern (MROP) testing technique.

The Metamorphic Relationship Output Pattern (MROP) is a testing technique primarily used when it is difficult to determine the correct output for a given input. This approach is based on defining the relationships between inputs and outputs, rather than focusing solely on the exact output values. If a certain relationship holds for the output of multiple test inputs, then it is assumed that the program is likely to be correct.

For a deeper understanding of the MROP technique, refer to the main paper titled "Detecting Hidden Failures of DBMS: A Comprehensive Metamorphic Relation Output Patterns Approach" published in the IEEE COMPSAC 2023 proceedings.

4. ### Incorrect zero value when using COALESCE() and IFNULL()
    - **Description:** When using `COALESCE()` and `IFNULL()` functions on a `BIGINT` datatype and casting into `UNSIGNED`, an incorrect zero value is returned.
    - ** MROP:** Subbag equality
    - **Test Case:**
    ```sql
	CREATE TABLE t0(c0 BIGINT UNSIGNED);
	INSERT INTO t0(c0) VALUES(NULL);
	SELECT * FROM t0 WHERE CAST(COALESCE(t0.c0, -1) AS UNSIGNED); -- expected: row is selected, actual: no row is selected
	SELECT * FROM t0 WHERE CAST(IFNULL(t0.c0, -1) AS UNSIGNED); -- expected: row is selected, actual: no row is selected
    ```
